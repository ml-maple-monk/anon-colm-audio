# Audio Comparison Page

Static page comparing clean vs degraded audio with ASR transcriptions.

## Local preview

```bash
cd exact/prepare_samples

# Rebuild index.html from sample data
python page/build.py --input page/data/selected_v3_human_eval

# Serve (must run from exact/prepare_samples/ so audio paths resolve)
python -m http.server 8080
```

Open: `http://localhost:8080/page/index.html`

On WSL2, use the WSL2 IP from the Windows host:
```bash
ip addr show eth0 | grep 'inet ' | awk '{print $2}' | cut -d/ -f1
# → e.g. 172.24.142.19
```
Then open: `http://172.24.142.19:8080/page/index.html`

---

## Rebuild after adding or editing samples

```bash
cd exact/prepare_samples
python page/build.py --input page/data/selected_v3_human_eval
```

`build.py` walks `page/data/selected_v3_human_eval/*/`, reads each `meta.json`, and embeds all sample data into `page/index.html` as a self-contained static file.

## Deploy to GitHub Pages

### 1. Ensure the right files are committed

The `page/` directory must contain:
```
page/
  index.html          ← generated by build.py
  build.py
  README.md
  data/
    selected_v3_human_eval/
      clipping__en__0.4__fleurs__rank01/
        clean.wav
        degraded.wav
        meta.json
      ...
```

Commit everything under `page/`:
```bash
git add exact/prepare_samples/page/
git commit -m "add audio comparison page with samples"
git push
```

### 2. Enable GitHub Pages

In the repository on GitHub:

1. Go to **Settings → Pages**
2. Under **Source**, select **Deploy from a branch**
3. Branch: `main` (or your branch), Folder: `/ (root)`
4. Click **Save**

### 3. Set the correct subfolder URL

GitHub Pages serves from the repo root, so the page is at:

```
https://<username>.github.io/<repo>/exact/prepare_samples/page/
```

Audio files resolve as `data/selected_v3_human_eval/...` relative to `page/`, so they load correctly with no changes needed.

### Notes

- Audio files (~10 MB total) are well under GitHub's 100 MB per-file and 1 GB repo limits — no Git LFS required.
- `index.html` is fully self-contained (no external CDN dependencies).
- Re-run `build.py` and commit the new `index.html` whenever samples change.
